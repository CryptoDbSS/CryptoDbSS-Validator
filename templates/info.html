This software implements the use of ECDSA Secp256k1 cryptographic algorithms based on generating signatures and deriving public keys using a private key, the secret key generates signatures in x509 format and derives the public key in p1363 format to authenticate transactions from the Front-end without the need to compromise the private key over the network.

The Back-End is written in C++ and has its own blockchain binary database model, each transaction is verified with the signature of the user's public key in addition to the implementation of data integrity verification algorithms such as SHA3-256 and sums between previous transactions to verify that the values are correct.

This app is scalable by nature due to the design in the identification of block and transaction type, which allows adding new features in the future such as smart contracts, internal tokens or any function that you want and/or wish to add.

It can be configured in a network with other nodes that run the app, a unique consensus algorithm was developed for this app that is based on which ledger node is algebraically closest to the hash of the last block, this in turn makes it more efficient than blockchains that require mining, being efficient in energy and performance.
